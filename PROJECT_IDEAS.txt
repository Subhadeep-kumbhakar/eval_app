1. CORE REQUIREMENTS (From User)
================================================================================

A) TEACHER SIDE
   - Upload one or multiple PDFs (course material, textbooks, notes)
   - Set total marks for the exam
   - Set weightage per topic (e.g., Chapter 1 = 30%, Chapter 2 = 20%)
   - Specify question distribution:
       * Number of MCQ questions
       * Number of Fill-in-the-Blanks questions
       * Number of Subjective/Descriptive questions
   - System auto-generates a question paper from uploaded content
   - Evaluate student answer sheets
   - Set evaluation strictness: Easy / Medium / Hard
   - Assign final marks to students

B) STUDENT SIDE
   - Separate registration/login
   - Submit answer sheets (text-based or PDF upload)
   - View marks and feedback after evaluation

C) TECH STACK
   - Python (backend)
   - Streamlit (frontend UI)
   - LLM (question generation + answer evaluation)
   - RAG (Retrieval-Augmented Generation) for context-aware generation
   - Similarity Search (vector-based answer matching)

================================================================================
2. ADDITIONAL IDEAS & ENHANCEMENTS (Suggested)
================================================================================

A) SMART QUESTION GENERATION
   - Bloom's Taxonomy tagging: Tag each question with cognitive level
     (Remember, Understand, Apply, Analyze, Evaluate, Create)
   - Difficulty level per question (Easy / Medium / Hard)
   - Duplicate detection: Ensure no repeated/similar questions
   - Answer key auto-generation: Generate model answers alongside questions
   - Distractor generation for MCQs: LLM generates plausible wrong options

B) ADVANCED EVALUATION
   - Rubric-based grading: Teacher defines rubric, LLM follows it
   - Partial marking: Award partial credit for subjective answers
   - Keyword matching + Semantic similarity (hybrid scoring)
   - Confidence score: Show how confident the system is in its grading
   - Detailed feedback generation: Per-question feedback for students

C) ANALYTICS & REPORTING
   - Class performance dashboard (avg scores, pass %, topic-wise analysis)
   - Per-student performance report
   - Question difficulty analysis (post-exam: which questions were hard)
   - Export results as CSV/PDF

D) QUESTION BANK
   - Save generated questions to a reusable question bank
   - Tag questions by topic, difficulty, type
   - Teachers can review/edit/approve questions before finalizing

E) SECURITY & INTEGRITY
   - Plagiarism detection between student answers (cosine similarity)
   - Time-stamped submissions
   - Role-based access (teacher vs student)

F) QUALITY OF LIFE
   - Multi-format PDF support (scanned PDFs via OCR using Tesseract)
   - Bulk student upload via CSV
   - Email notifications for results (optional)
   - Session management for exams

3. PROPOSED TECH STACK (Detailed)
================================================================================

Layer               | Technology
--------------------|-------------------------------------------
Frontend            | Streamlit
Backend/Logic       | Python
LLM                 | OpenAI GPT-4 / Google Gemini / Ollama (local)
Embeddings          | sentence-transformers / OpenAI embeddings
Vector Store        | ChromaDB (lightweight, local) or FAISS
PDF Parsing         | PyPDF2 + pdfplumber (text) / Tesseract (OCR)
RAG Framework       | LangChain
Database            | SQLite (simple) or PostgreSQL (production)
Auth                | Streamlit-authenticator or custom session
Export              | ReportLab (PDF) / pandas (CSV)
